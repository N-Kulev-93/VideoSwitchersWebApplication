# About 
As part of my preparation for a job interview I want to create a couple of projects starting with this project serving as template for other projects. I want to mention that I have not practiced proffesionally some of the challenges that SPA applications carry, so I will be actively learning in this process. I 100% open for discussion with anyone who find problems in my implementation and wants to help. To summarize goal of this project is to learn, show my coding style plus some basic skills by providing nessercery setup for a single page application to build up on.
I will mention some of the features im planning to support using selected technologies:
- **Ready-to-Use and Learn project** - working project template for single page applications development and helping other people who want to fill knowledge gaps
- **Easy development environment** - Webpack 's dev server setup (client hot-reload, in-memory build, optimized bundle), Jest setup (client unit testing), ESLint setup (client friendly compiler), ASP.Net nUnit (server unit testing), OpenAPI standart compliant controller to serve as example for projects using the template.
- **Easy deployment environment** - Docker image multi-stage build and push on merge using GitHub Actions

This project is suitable for non-public web applications like internal tools and dashboards with high interactivity.
This project is NOT suitable for content-rich web applications that require SEO oriented implementation.
